## Overview

Tyk provides three different ways to install and manage your API Gateway on Kubernetes, depending on your deployment needs. Choose the option that best fits your architecture:

<ResponsiveGrid>

<Card href="/tyk-self-managed/install/kubernetes/helm-self-managed" title="Helm Self-Managed">
Full Tyk stack (Gateway, Dashboard, Pump, Developer Portal) deployed via Helm.
</Card>

<Card href="/tyk-self-managed/install/kubernetes/helm-hybrid" title="Helm Hybrid">
Tyk Data Plane (Gateway, Pump) deployed via Helm, connecting to a Tyk Cloud Control Plane.
</Card>

<Card href="/tyk-self-managed/install/kubernetes/standalone-operator" title="Standalone Operator">
Tyk Operator deployed via Helm to manage APIs declaratively.
</Card>
