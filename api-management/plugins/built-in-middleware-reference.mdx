This page provides a comprehensive reference of all built-in middleware in Tyk, their execution stages, and data access capabilities.

## Execution Stages

Tyk processes API requests through a series of stages. Middleware can be injected at different points in this lifecycle:

- **Pre**: Executed before any authentication or rate limiting occurs.
- **AuthCheck**: Executed during the authentication phase.
- **PostKeyAuth**: Executed immediately after successful authentication.
- **Post**: Executed after authentication and rate limiting, before the request is proxied to the upstream.
- **Response**: Executed on the response returned from the upstream before it is sent back to the client.

## Data Access Capabilities

Middleware can interact with different types of data during the request lifecycle:

- **Session**: Access to the authenticated user's session object.
- **Context**: Access to the request context variables.
- **Metadata**: Access to the API or session metadata.
