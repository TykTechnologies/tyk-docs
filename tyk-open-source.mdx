## What is Tyk Open Source

Open source is at the heart of what we do. Anything that is API Gateway-related lives in the Gateway, or is critical for the Gateway to work is open and freely available via our [Github](https://github.com/TykTechnologies/tyk).

The Tyk Gateway is fully open-source.  It's all the same Gateway that's used by you (the community!), by our enterprise products, as well as our SaaS.

Our commitment to open source also delivers a host of benefits for our users: sign up for free with Tyk, receive securely packaged open source packages, get started guides, access to our community and all of the latest open source information.

## Tyk Open Source Compared to Tyk Self-Managed and Tyk Cloud

The table below summarizes which Tyk components are available in Tyk Open Source (OSS) compared to the licensed [Tyk Self-Managed](/tyk-self-managed) and [Tyk Cloud](/tyk-cloud) editions. Use this as a starting point when planning a deployment; follow the linked pages for full details on each component.

| Component | Tyk Open Source | Tyk Self-Managed | Tyk Cloud |
| :-- | :-- | :-- | :-- |
| [Tyk Gateway](/tyk-oss-gateway) — REST, GraphQL, TCP, gRPC, rate limiting, authentication, transforms | Included | Included | Included |
| [Tyk Pump](/api-management/tyk-pump) — analytics exporter to Prometheus, Elasticsearch, and other backends | Included | Included | Managed |
| [Tyk Identity Broker (TIB)](/tyk-identity-broker/overview) — bridge to third-party IdPs (LDAP, social OAuth, SAML) | Included | Included | Managed |
| [Tyk Helm Charts](/product-stack/tyk-charts/overview) — Kubernetes deployment charts | Included | Included | N/A |
| [Tyk Dashboard](/api-management/dashboard-configuration) — GUI, users and RBAC, analytics dashboards | Not available | Included | Included |
| [Classic Developer Portal](/tyk-developer-portal/tyk-portal-classic) — API catalog, developer self-service | Not available (requires Dashboard) | Included | Included |
| [Tyk Enterprise Developer Portal](/portal/overview/intro) — customizable developer portal | Not available | Add-on | Add-on |
| [Tyk Multi-Data Centre Bridge (MDCB)](/api-management/mdcb) — multi-region and edge Gateways | Not available | Add-on | Managed |
| [Tyk Operator](/api-management/automations/operator) — Kubernetes-native API management (GitOps) | Community edition available (`TYK_MODE=ce`) | Included | Included |
| [Tyk Sync](/api-management/automations/sync) — sync API definitions, policies, and templates to Git | Not available from Sync v2.0 onwards (see [Tyk Sync](/api-management/automations/sync)) | Included | Included |

<Note>
Feature availability can vary by release. For version-specific capabilities, check the [release notes](/developer-support/release-notes/gateway) for the component you are deploying. Some add-ons (MDCB, Enterprise Developer Portal) require a separate license even on Tyk Self-Managed and Tyk Cloud — contact your Tyk account team for details.
</Note>

If you are evaluating whether to run Tyk Open Source or a licensed edition, the general rule is:

- Choose **Tyk Open Source** when you only need the Gateway (with optional Pump, TIB, and Helm charts) and are comfortable managing APIs, policies, and keys through the [Gateway API](/tyk-gateway-api) or file-based configuration.
- Choose **Tyk Self-Managed** or **Tyk Cloud** when you need the Dashboard GUI, a developer portal, multi-region deployments (MDCB), or GitOps with [Tyk Sync](/api-management/automations/sync).
