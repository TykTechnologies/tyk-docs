In this tutorial, you will learn how to use the credentials from an App you've created in Tyk AI Studio to build a simple command-line interface (CLI) application with NodeJS. This application will take a text prompt from you, send it to an LLM via the Tyk AI Gateway, and print the response.

## Prerequisites

Before you begin, you will need:
-   **An approved App in Tyk AI Studio:** You should have already created an App and had it approved by a Studio Administrator. If you haven't, please follow the guide on [App Management for Consumers](/ai-management/ai-studio/consumer-apps).
-   **Your App's Key ID and Secret:** You will need these to authenticate your application.
-   **Node.js and npm installed:** This tutorial uses NodeJS. You can download it from [nodejs.org](https://nodejs.org/).
-   **An LLM configured in AI Studio:** Your App must be configured to use at least one LLM (e.g., OpenAI's GPT-4).

## Instructions

### Step 1: Set up your NodeJS project

First, create a new folder for your project and initialize a new NodeJS project within it.

```bash
mkdir tyk-ai-cli-app
cd tyk-ai-cli-app
npm init -y
```

Next, you'll need a way to make HTTP requests. We'll use the popular `axios` library for this.

```bash
npm install axios
```

### Step 2: Create the CLI application file

Create a new file named `index.js` in your project folder. This is where you will write the code for your CLI application.

```bash
touch index.js
```

### Step 3: Write the application code

Open `index.js` in your favorite code editor and add the following code. Be sure to replace the placeholder values with your actual App credentials and AI Gateway endpoint.

```javascript
const axios = require('axios');
const readline = require('readline');

// --- Configuration ---
// Replace these with your actual credentials and endpoint
const AI_GATEWAY_ENDPOINT = 'https://your-ai-gateway-endpoint.com/openai/v1/chat/completions';
const KEY_ID = 'YOUR_APP_KEY_ID';
const SECRET = 'YOUR_APP_SECRET';
// -------------------

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

async function getAiResponse(prompt) {
  try {
    const response = await axios.post(
      AI_GATEWAY_ENDPOINT,
      {
        model: 'gpt-4', // Or whichever model your App is configured to use
        messages: [{ role: 'user', content: prompt }]
      },
      {
        headers: {
          'Authorization': `Bearer ${KEY_ID}:${SECRET}`,
          'Content-Type': 'application/json'
        }
      }
    );

    if (response.data.choices && response.data.choices.length > 0) {
      return response.data.choices[0].message.content;
    } else {
      return 'Sorry, I could not get a response.';
    }
  } catch (error) {
    console.error('Error calling AI Gateway:', error.response ? error.response.data : error.message);
    return 'An error occurred.';
  }
}

function startCli() {
  rl.question("Hi, what's your name? ", async (prompt) => {
    if (prompt.toLowerCase() === 'exit') {
      rl.close();
      return;
    }

    console.log('...thinking...');
    const aiResponse = await getAiResponse(prompt);
    console.log(`AI: ${aiResponse}`);
    
    // Ask the next question
    askQuestion();
  });
}

function askQuestion() {
    rl.question('> ', async (prompt) => {
        if (prompt.toLowerCase() === 'exit') {
            rl.close();
            return;
        }

        console.log('...thinking...');
        const aiResponse = await getAiResponse(prompt);
        console.log(`AI: ${aiResponse}`);
        askQuestion();
    });
}

console.log('Connected to AI Studio. Ask me anything! (type "exit" to quit)');
startCli();

```

**A note on the code:**
- It uses the `readline` module to get input from the command line.
- The `getAiResponse` function makes a `POST` request to the AI Gateway's OpenAI-compatible endpoint.
- It sets the `Authorization` header with your App's Key ID and Secret, in the format `Bearer ${KEY_ID}:${SECRET}`.
- It sends a simple prompt to the model and prints the response.

### Step 4: Run your CLI App

Now you can run your application from your terminal.

```bash
node index.js
```

The application will start and ask for your name. You can then have a conversation with the LLM. To quit the application, type `exit`.

```
Connected to AI Studio. Ask me anything! (type "exit" to quit)
Hi, what's your name? > My name is Alex.
...thinking...
AI: Hello Alex, it's nice to meet you! How can I help you today?
> What is Tyk?
...thinking...
AI: Tyk is an open-source API Gateway that is lightweight, performant, and cloud-native.
> exit
```

Congratulations! You have successfully used your Tyk AI Studio App credentials to build a working AI-powered CLI application.

## Frequently Asked Questions

<AccordionGroup>
<Accordion title="I'm getting a 401 Unauthorized error. What's wrong?">
A 401 error means your request is not properly authenticated. Check the following:
-   Make sure your App has been approved by an administrator and its credentials are active.
-   Double-check that you have correctly copied your Key ID and Secret into the `index.js` file.
-   Ensure your `Authorization` header is in the correct format: `Bearer KEY_ID:SECRET`.
</Accordion>
<Accordion title="Can I use a different LLM?">
Yes. You can change the `model` parameter in the `axios.post` call to any model that your App is configured to use. You may also need to change the `AI_GATEWAY_ENDPOINT` if you are not using an OpenAI-compatible model. Check the documentation for your specific LLM provider in AI Studio.
</Accordion>
</AccordionGroup>
