After completing the [quickstart](/ai-management/mcp-gateway/quickstart), you have a working MCP proxy — but it accepts connections from any client. This guide secures your remote MCP server so that only agents with a valid key can reach it.

---

## Before you begin

- A Tyk Gateway (v5.13 or later) connected to your Tyk Dashboard
- The Mock MCP Server running on `http://localhost:7878` — see the [quickstart](/ai-management/mcp-gateway/quickstart)
- An MCP proxy named **Mock MCP Server** already created — also covered in the quickstart
- Node.js 18 or later (to run MCP Inspector)
- A Dashboard user account with MCP write permissions

---

## Step 1: Enable authentication

Open the proxy in the Tyk Dashboard. In the sidebar, click **MCP**, then click **Edit** next to **Mock MCP Server**.

In the designer, click the **Authentication** switch.

Select **Auth Token** as the Authentication method.

Set the Authentication token location to **use header value** and leave the header name as `Authorization`.

![Auth token header configuration](/img/ai-management/mcp-how-to-secure-add-header.png)

Click **Save MCP Proxy**.

The proxy now requires a bearer token on every request. Clients that connect without a valid key receive a `401 Unauthorized` response.

---

## Step 2: Issue an API key

In the Dashboard sidebar, click **Keys**, then click **Add Key**.

Under **Access rights**, click **Choose API** and select **Mock MCP Server**.

Click **Create Key**. Copy the key shown — you cannot retrieve it after navigating away.

![API key created](/img/ai-management/mcp-how-to-secure-add-key.png)

---

## Step 3: Verify with MCP Inspector

Start MCP Inspector:

```bash
npx @modelcontextprotocol/inspector
```

Open the URL printed in your terminal. In the Inspector UI:

1. Set **Transport Type** to `Streamable HTTP`
2. Set **URL** to your MCP endpoint (find it under **MCP Proxy URL** in the proxy designer, then append `/mcp`)
3. Click **Connect** — without adding a header

The connection fails. MCP Inspector reports a `401 Unauthorized` error. Authentication is working.

Now add the key and try again:

1. Add a header: `Authorization` = `Bearer {your-api-key}`
2. Click **Connect**

![MCP Inspector connected with API key](/img/ai-management/mcp-how-to-secure-mcp-inspector.png)

The connection succeeds. Click the **Tools** tab — all 15 Mock MCP Server tools appear.

---

## Limitations and alternatives

API key authentication via a bearer token header is a straightforward way to secure an MCP proxy, but it has limitations — keys are long-lived, there is no built-in token expiry or rotation, and clients must manage the key securely.

For more demanding scenarios, Tyk supports a range of [client authentication methods](/api-management/client-authentication), including JWT, mutual TLS, and OAuth 2.1. For MCP specifically, Tyk extends OAuth 2.1 with Protected Resource Metadata so MCP-aware clients can discover authentication requirements automatically — see [MCP Gateway — OAuth 2.1 authentication](/ai-management/mcp-gateway/oauth-2-1).

---

## What's next

API key authentication is one layer of security. Tyk provides several more that you can combine on the same proxy:

- **Restrict which tools an agent can call** — Configure a tool allowlist so agents can only call approved tools, and new tools added by the upstream are blocked by default. See [MCP middleware](/ai-management/mcp-gateway/mcp-middleware).
- **Limit request volume** — Apply rate limits per key to protect expensive tools from overuse. See [How to rate limit individual MCP tools per consumer](/ai-management/mcp-gateway/how-to-rate-limit-tools-per-consumer).
- **Separate access tiers** — Use policies to give different agents different levels of access. See [MCP proxy policies](/ai-management/mcp-gateway/policies).
- **OAuth 2.1** — For MCP-aware clients that support automatic auth discovery, configure Protected Resource Metadata so clients can find Tyk's auth requirements without manual configuration. See [MCP Gateway — OAuth 2.1 authentication](/ai-management/mcp-gateway/oauth-2-1).
