An API Provider is a connection configuration between the Tyk Developer Portal and a Tyk Dashboard instance. Each Provider stores the credentials and settings the Portal uses to discover APIs, enforce access policies, and issue credentials to API consumers.

## Provider Architecture

The Portal connects to a Tyk Dashboard through its management API. Through this connection, the Portal can:

- **Discover resources** — query APIs and access policies available in the Dashboard
- **Create policies** — generate access and rate-limit policies when API Products and Plans are created
- **Issue credentials** — request API keys, OAuth tokens, or other credential types when consumers request access
- **Synchronize changes** — pull policy updates from the Dashboard to keep the Portal catalog in sync

## Multi-Provider Support

A single Portal can connect to multiple Tyk Dashboard instances simultaneously. Each connection is a separate Provider record. Common use cases include:

- **Environment segregation** — connect to separate development, staging, and production Dashboards
- **Organizational boundaries** — connect to Dashboards managed by different teams or departments
- **Geographic distribution** — connect to Dashboards in different regions or data centers
- **Migration paths** — facilitate transitions between Tyk deployments without downtime

## Provider Limitations

Each API Product or Plan is associated with a single Provider:

- Credentials issued by one Provider cannot be used with APIs from a different Provider.
- All APIs within an API Product must use the same authentication method, so that a single credential works across the entire Product.

## Dashboard User Requirements

The Portal connects to each Tyk Dashboard using a dedicated Dashboard user account. Before adding a Provider, create this user in the Tyk Dashboard and grant it the following permissions:

| Permission   | Access level |
| :----------- | :----------- |
| APIs         | Write        |
| Certificates | Write        |
| Keys         | Write        |
| Policies     | Write        |
| Analytics    | Read         |
| Users        | Read         |

See [Manage Tyk Dashboard Users](/api-management/user-management#manage-tyk-dashboard-users) for steps.

<Note>
Use a dedicated Dashboard user for each Portal connection. Sharing credentials with other systems complicates access auditing and secret rotation.
</Note>
