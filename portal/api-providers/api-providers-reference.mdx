Reference for all API Provider configuration fields, read-only metadata, and Portal API management endpoints.

## Provider Fields

### Provider Name

- **Location:** *Providers > Add/Edit Provider > Name*
- **Purpose:** Identifies this Provider in the Admin Portal and in sync result reports.
- **Best practice:** Use a name that indicates the environment or purpose, for example `Production APIs` or `Staging Dashboard`.

### Connection URL

- **Location:** *Providers > Add/Edit Provider > URL*
- **Format:** Full URL including protocol, host, and port. Example: `https://dashboard.example.com:3000`
- **Note:** Must be the Tyk **Dashboard** URL, not the Tyk Gateway URL. The Portal uses this address to call the [Tyk Dashboard API](/api-management/dashboard-configuration#exploring-the-dashboard-api).
- **Requirement:** Must be reachable from the Portal server.

### Secret

- **Location:** *Providers > Add/Edit Provider > Secret*
- **Format:** The Tyk Dashboard API Access Credentials for a Dashboard user account.
- **Best practice:** Use a dedicated Dashboard user with [minimum required permissions](/portal/api-providers/what-are-api-providers#dashboard-user-requirements).

### Organisation ID

- **Location:** *Providers > Add/Edit Provider > Organisation ID*
- **Purpose:** Scopes the Provider's access to a specific Tyk organisation's resources in multi-tenant deployments.
- **Where to find it:** Tyk Dashboard user profile.
- **Default:** Leave empty if your deployment has a single Tyk organisation.
- **Note:** This is not the same as a Developer Portal [Organisation](/portal/overview/concepts#api-consumers).

### Insecure Skip Verify

- **Location:** *Providers > Add/Edit Provider > Insecure skip verify*
- **Options:**
  - `Enabled` — bypasses TLS certificate validation
  - `Disabled` — enforces strict TLS certificate validation (default)
- **Security note:** When enabled, the connection does not verify the Provider's identity through its certificate and is vulnerable to man-in-the-middle attacks. Use only in development or test environments with self-signed certificates.

### Policy Tags

- **Location:** *Providers > Add/Edit Provider > Policy tags*
- **Format:** Comma-separated list of text tags with no spaces after the comma. Example: `tag1,tag2`
- **Behavior:**
  - The configured tags are automatically applied to all Tyk Dashboard policies created by the Portal (for API Products and Plans).
  - During sync, only Dashboard policies that carry these tags are retrieved.
  - Adding these tags to an existing partitioned policy in the Dashboard causes it to be retrieved in the next sync, creating a placeholder API Product or Plan.
- **Version note:** Tag propagation behavior was updated in v1.14.0. On earlier versions, manually created Plans and Products could disappear after sync because tags were not consistently applied to the corresponding Dashboard policies. Upgrade to v1.14.0 or later.

### Baseline URL

- **Location:** *Providers > Add/Edit Provider > Connection tab*
- **Format:** Full URL including protocol and host. Example: `https://gateway.example.com`
- **Purpose:** The root URL of the Tyk Gateway where APIs from this Provider can be accessed. Displayed on Product pages in the Live Portal.
- **v1.16+:** Used to auto-populate the GraphQL server URL when adding a GraphQL API to an API Product.

## Read-Only Provider Metadata

These fields are displayed in the Admin Portal and returned by the Portal API but cannot be configured directly.

| Field | Description |
| :---- | :---------- |
| **Type** | Provider type. Always `tyk-pro` in the current release. |
| **Status** | Current connection status of the Provider. |
| **Last Synced** | Timestamp of the last successful synchronization with the Dashboard. |
| **Product Count** | Number of API Products associated with this Provider. |
| **Plan Count** | Number of Plans associated with this Provider. |
| **API Count** | Number of APIs fetched from this Provider. Available since v1.13.0. |

## Portal API Endpoints

Use the Portal REST API to manage Providers programmatically.

| Operation | Method | Endpoint |
| :-------- | :----- | :------- |
| List all Providers | `GET` | `/portal-api/providers` |
| Create a Provider | `POST` | `/portal-api/providers` |
| Update a Provider | `PUT` | `/portal-api/providers/{id}` |
| Delete a Provider | `DELETE` | `/portal-api/providers/{id}` |
| Sync a specific Provider | `PUT` | `/portal-api/providers/{id}/action/Synchronize` |
| Sync all Providers | `PUT` | `/portal-api/providers/action/Synchronize` |

See the [Portal API reference](/product-stack/tyk-enterprise-developer-portal/api-documentation/list-of-endpoints/portal-api-list-of-endpoints) for authentication requirements and request/response schemas.

## Configuration Reference

### PORTAL_REFRESHINTERVAL

Controls the interval between automatic Provider synchronizations.

| Setting | Value |
| :------ | :---- |
| Type | Integer |
| Default | `10` |
| Unit | Minutes |

<Warning>
Do not set `PORTAL_REFRESHINTERVAL` to `0`. The Portal will fail to start.
</Warning>

See [PORTAL_REFRESHINTERVAL](/product-stack/tyk-enterprise-developer-portal/deploy/configuration#portal_refreshinterval) in the Configuration Reference for the complete specification.
