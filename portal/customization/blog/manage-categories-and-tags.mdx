## Categories and Tags

Categories and tags both organize blog posts, but they serve different purposes:

| | Categories | Tags |
|---|---|---|
| **Purpose** | Group posts by broad topic | Link posts to API Products and enable cross-referencing |
| **Structure** | Flat (no nesting supported) | Flat |
| **Live Portal** | Filter page at `/blog/category/[name]` | Related blog content on API Product pages |
| **Naming rules** | Any name | Lowercase, alphanumeric characters and `-`, `_`, `.`, `/` only — no spaces |

## Create a Blog Category

1. **Open the categories section.** In the Admin UI sidebar, select **Blog**, then click **Categories**.

   {/* TODO: Screenshot — Admin UI Blog > Categories list page */}

1. **Start a new category.** Click **Add category**.

1. **Enter the category name.** Type a name in the **Name** field. Category names have no character restrictions and can include spaces.

1. **Save the category.** Click **Save**.

The category is now available in the **Categories** field when creating or editing a blog post.

Categories appear in the Live Portal navigation under `/blog/category/[category-name]`. Visitors can browse all posts assigned to a category from this URL.

## Assign a Category to a Post

1. Open or create a blog post (see [Create and Manage Blog Posts](/portal/customization/blog/manage-blog-posts)).
1. In the **Categories** field, select one or more categories.
1. Save the post.

A post can belong to multiple categories simultaneously.

## Create a Blog Tag

1. **Open the tags section.** In the Admin UI sidebar, select **Blog**, then click **Tags**.

   {/* TODO: Screenshot — Admin UI Blog > Tags list page */}

1. **Start a new tag.** Click **Add tag**.

1. **Enter the tag name.** Type a name in the **Name** field.

   <Warning>
   Tag names are validated strictly. They must be lowercase and contain only alphanumeric characters and the following punctuation: `-`, `_`, `.`, `/`. Spaces are not allowed. Tag names are automatically converted to lowercase on save.
   </Warning>

   Valid examples: `api-gateway`, `v1.0`, `rest/http`

   Invalid examples: `API Gateway` (uppercase and spaces), `rest api` (space)

1. **Save the tag.** Click **Save**.

The tag is now available in the **Tags** field when creating or editing a blog post.

## Assign Tags to a Post

1. Open or create a blog post (see [Create and Manage Blog Posts](/portal/customization/blog/manage-blog-posts)).
1. In the **Tags** field, select one or more tags.
1. Save the post.

A post can have multiple tags.

## Link Blog Posts to API Products

Tags create a connection between blog posts and API Products. When a blog post shares a tag with an API Product, that post appears in the **Related blog content** section on the product's page in the Live Portal.

To enable this:

1. Create a tag that describes the relevant API or topic (for example, `payments-api`).
1. Assign that tag to the relevant blog posts.
1. Assign the same tag to the API Product in its product settings.

Posts with matching tags automatically appear in the **Related blog content** section on the product page — no further configuration is required.

## Delete a Category or Tag

Deleting a category or tag removes it from the Admin UI lists and from any posts it was assigned to. Posts themselves are not deleted.

To delete:
1. Go to **Blog** > **Categories** (or **Tags**).
1. Select the item to delete.
1. Click **Delete** and confirm.
