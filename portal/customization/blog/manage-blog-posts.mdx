| Component | Version | Edition |
|---|---|---|
| Tyk Enterprise Developer Portal | Available since v1.0.0 | Enterprise (license must include blog capability) |

## Prerequisites

- The blog is enabled in your portal configuration (`Blog.Enable: true` / `PORTAL_BLOG_ENABLE=true`). See [Blog Posts Overview](/portal/customization/blog/overview) for configuration details.
- Admin access to the Developer Portal Admin UI.

## Create a Blog Post

1. **Open the blog section.** In the Admin UI sidebar, select **Blog**, then click **Posts**.

   {/* TODO: Screenshot — Admin UI sidebar with Blog > Posts highlighted */}

1. **Start a new post.** Click **Add post**.

   {/* TODO: Screenshot — Blog Posts list page with Add post button visible */}

1. **Enter the post title.** In the **Title** field, type the title of your post.

1. **Add a summary.** In the **Lede** field, enter a short summary or introductory paragraph. The Lede is typically displayed on the blog listing page alongside the post title.

1. **Choose your editor.** Toggle **Display Markdown** to switch between the rich text editor and the Markdown editor.

   - When the toggle is off, use the **Content** field to write in rich text or HTML.
   - When the toggle is on, use the **Markdown Content** field to write in Markdown.

   See [Markdown in Developer Portal](/portal/customization/markdown) for supported Markdown syntax and limitations.

1. **Upload a header image.** Click **Header image** and select an image file to use as the featured image for the post. The header image is rendered by the `blog_listing.tmpl` and `blog_detail.tmpl` templates via `{{ .HeaderImage.URL }}`.

1. **Set the URL path.** In the **Path** field, enter the URL slug for the post. This becomes part of the public URL: `/blog/YYYY/M/D/[path]`, where the date is derived from when the post is created.

   <Note>
   The publish date in the post URL is set at creation time from the post's creation timestamp. There is no separate date picker — if you need the post to appear under a specific date, create it on that date.
   </Note>

1. **Assign categories.** Select one or more categories from the **Categories** field. Categories group your posts by topic and appear as filters in the Live Portal. To create categories first, see [Create and Manage Blog Categories and Tags](/portal/customization/blog/manage-categories-and-tags).

1. **Assign tags.** Select one or more tags from the **Tags** field. Tags control which blog posts appear in the **Related blog content** section on API Product pages in the Live Portal. To create tags first, see [Create and Manage Blog Categories and Tags](/portal/customization/blog/manage-categories-and-tags).

1. **Set the post status.** In the **Status** field, choose one of:

   | Status | Description |
   |---|---|
   | `draft` | Not visible in the Live Portal. Use for posts still being written. |
   | `review` | Not visible in the Live Portal. Use for posts awaiting approval. |
   | `published` | Visible in the Live Portal at the post's URL. |

1. **Save the post.** Click **Save** to create the post.

   {/* TODO: Screenshot — Completed new blog post form before saving */}

When the post status is set to `published`, it becomes visible at `/blog/YYYY/M/D/[path]` in the Live Portal and appears on the blog listing page at `/blog`.

## Edit a Blog Post

1. In the Admin UI, go to **Blog** > **Posts**.
1. Click the title of the post you want to edit.
1. Make your changes to any field.
1. Click **Save**.

Changes to a published post are reflected immediately in the Live Portal.

## Change a Post's Status

To unpublish a post, edit the post and change its **Status** from `published` to `draft` or `review`. The post is removed from the Live Portal immediately.

To republish it, set the status back to `published`.

## Delete a Blog Post

1. In the Admin UI, go to **Blog** > **Posts**.
1. Select the post you want to delete.
1. Click **Delete** and confirm.

Deleting a post is permanent. The post URL returns a 404 in the Live Portal.
