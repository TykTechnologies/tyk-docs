The Tyk Enterprise Developer Portal is available as a Docker container, Helm chart, or RPM package. This guide explains how to install and bootstrap the portal.

## System Requirements

Before installing the portal, ensure your environment meets the following requirements.

### Operating System Requirements

<Warning>
**Important: GLIBC Compatibility**

Starting with Portal v1.16.0, the portal binary requires GLIBC 2.34 or later. This means:
- **Ubuntu 22.04 (Jammy) or later** is required
- **Ubuntu 20.04 is NOT supported** - it ships with GLIBC 2.31
- **RHEL/CentOS 9** or later is required
- **Debian 12 (Bookworm)** or later is required

If you attempt to run Portal v1.16.0+ on an older OS, you will encounter errors like:
```
/opt/portal/dev-portal: /lib/x86_64-linux-gnu/libc.so.6: version `GLIBC_2.34' not found
```

**Solution:** Upgrade your host OS to a supported version or use the Docker installation method.
</Warning>

### Supported Operating Systems

| Operating System | Version | Notes |
|-----------------|---------|-------|
| Ubuntu | 22.04 (Jammy) or later | Recommended for Linux installations |
| Red Hat Enterprise Linux | 9 or later | For RPM-based installations |
| CentOS Stream | 9 or later | For RPM-based installations |
| Debian | 12 (Bookworm) or later | - |
| Docker | Any host with Docker 20.10+ | Recommended installation method |
| Kubernetes | 1.19+ with Helm 3+ | For containerized deployments |

### Database Requirements

The portal requires a database to store metadata. Supported databases:

- **PostgreSQL** 12 or later (recommended)
- **MySQL** 8.0 or later
- **MariaDB** 10.6 or later

<Note>
**Note: SQLite support**

Tyk Developer Portal no longer supports SQLite as of version 1.14.0. Please use PostgreSQL or another supported database.
</Note>

## Enterprise Developer Portal Components

<img src="/img/dashboard/portal-management/enterprise-portal/portal-deployment-diagram.png" alt="Portal deployment diagram" />
<br/>

The portal deployment comprises three main components:
- **Portal application** - The main portal service
- **Database** - Stores metadata including API products, plans, developers, applications, and more
- **Asset storage** - Stores CMS assets such as images, themes, and OpenAPI specification files. Assets can reside in the database or separately in an S3 bucket or filesystem volume.

Optional components:
- **3rd party identity provider** - Required for [OAuth2.0 for API Products](/tyk-stack/tyk-developer-portal/enterprise-developer-portal/api-access/dynamic-client-registration)
- **[Tyk Identity Broker](/api-management/external-service-integration)** - Required for Single Sign-On configuration
- **Email server** - For sending notifications to admin users and developers

## Docker

This section explains how to install the Tyk Enterprise Developer Portal using Docker with PostgreSQL (recommended).

<Warning>
**Note**

This is an example configuration. Customize all credentials including username, password, and database name for your environment.
</Warning>

**Prerequisites**
- [Docker](https://docs.docker.com/get-docker/) 20.10 or later
- A valid Tyk Enterprise Portal license

### Installation Steps

1. **Create a network for the portal deployment**

    ```console
    docker network create tyk-portal
    ```

2. **Create an init script for PostgreSQL**

    Create a file named `init.sql`:
    ```sql
    -- init.sql
    CREATE USER admin WITH ENCRYPTED PASSWORD 'secr3t';
    CREATE DATABASE portal;
    GRANT ALL PRIVILEGES ON DATABASE portal TO admin;
    ```

3. **Create the database volume and launch PostgreSQL**

    ```console
    docker volume create tyk-portal-postgres-data
    ```

    ```console
    docker run \
    -d \
    --name tyk-portal-postgres \
    --restart on-failure:5 \
    -e POSTGRES_PASSWORD=secr3t \
    -e PGDATA=/var/lib/postgresql/data/pgdata \
    --mount type=volume,source=tyk-portal-postgres-data,target=/var/lib/postgresql/data/pgdata \
    --mount type=bind,src=$(pwd)/init.sql,dst=/docker-entrypoint-initdb.d/init.sql \
    --network tyk-portal \
    -p 5432:5432 \
    postgres:14-alpine
    ```

4. **Create an environment variables file**

    Create a file named `.env`:
    ```ini
    PORTAL_HOSTPORT=3001
    PORTAL_DATABASE_DIALECT=postgres
    PORTAL_DATABASE_CONNECTIONSTRING=host=tyk-portal-postgres port=5432 dbname=portal user=admin password=secr3t sslmode=disable
    PORTAL_DATABASE_ENABLELOGS=false
    PORTAL_THEMING_THEME=default
    PORTAL_STORAGE=db
    PORTAL_LICENSEKEY=<your-license-here>
    ```

    For all available configuration options, see the [configuration reference](/product-stack/tyk-enterprise-developer-portal/deploy/configuration).

5. **Pull and launch the portal container**

    Replace `<tag>` with the portal version (e.g., `v1.16`). See [Docker Hub](https://hub.docker.com/r/tykio/portal/tags) for available versions.
    ```console
    docker run -d \
        -p 3001:3001 \
        --env-file .env \
        --network tyk-portal \
        --name tyk-portal \
        tykio/portal:<tag>
    ```

6. **Bootstrap the portal**

    The portal is now running on port 3001. Complete the [bootstrapping process](#bootstrapping-enterprise-developer-portal) to create the super admin user.

7. **Clean up**

    To remove the portal installation:
    ```console
    docker stop tyk-portal tyk-portal-postgres
    docker rm tyk-portal tyk-portal-postgres
    docker volume rm tyk-portal-postgres-data
    ```

## Docker Compose

For development and testing, Docker Compose provides a simple way to run the portal with all dependencies.

**Prerequisites**
- [Docker](https://docs.docker.com/get-docker/) 20.10 or later
- [Docker Compose](https://docs.docker.com/compose/install/) v2 or later

### Installation Steps

1. **Create an init script for PostgreSQL**

    Create a file named `init.sql`:
    ```sql
    -- init.sql
    CREATE USER admin WITH ENCRYPTED PASSWORD 'secr3t';
    CREATE DATABASE portal;
    GRANT ALL PRIVILEGES ON DATABASE portal TO admin;
    ```

2. **Create an environment variables file**

    Create a file named `.env`:
    ```ini
    PORTAL_HOSTPORT=3001
    PORTAL_DATABASE_DIALECT=postgres
    PORTAL_DATABASE_CONNECTIONSTRING=host=tyk-portal-postgres port=5432 dbname=portal user=admin password=secr3t sslmode=disable
    PORTAL_DATABASE_ENABLELOGS=false
    PORTAL_THEMING_THEME=default
    PORTAL_LICENSEKEY=<your-license-here>
    PORTAL_STORAGE=db
    ```

3. **Create a docker-compose file**

    Create `docker-compose.yaml`. Replace `<tag>` with your desired portal version (e.g., `v1.16`):
    ```yaml
    version: '3.6'
    services:
      tyk-portal:
        depends_on:
          - tyk-portal-postgres
        image: tykio/portal:<tag>
        networks:
          - tyk-portal
        ports:
          - 3001:3001
        environment:
          - PORTAL_DATABASE_DIALECT=${PORTAL_DATABASE_DIALECT}
          - PORTAL_DATABASE_CONNECTIONSTRING=${PORTAL_DATABASE_CONNECTIONSTRING}
          - PORTAL_THEMING_THEME=${PORTAL_THEMING_THEME}
          - PORTAL_THEMING_PATH=${PORTAL_THEMING_PATH}
          - PORTAL_LICENSEKEY=${PORTAL_LICENSEKEY}
          - PORTAL_STORAGE=${PORTAL_STORAGE}

      tyk-portal-postgres:
        image: postgres:14-alpine
        volumes:
          - tyk-portal-postgres-data:/var/lib/postgresql/data/pgdata
          - ${PWD}/init.sql:/docker-entrypoint-initdb.d/init.sql
        networks:
          - tyk-portal
        environment:
          - POSTGRES_PASSWORD=secr3t
          - PGDATA=/var/lib/postgresql/data/pgdata

    volumes:
      tyk-portal-postgres-data:

    networks:
      tyk-portal:
    ```

4. **Launch the portal**

    ```console
    docker-compose --env-file .env up -d
    ```

5. **Bootstrap the portal**

    The portal is now running on port 3001. Complete the [bootstrapping process](#bootstrapping-enterprise-developer-portal) to create the super admin user.

6. **Clean up**

    ```console
    docker-compose down
    ```

## Kubernetes

Install the Tyk Enterprise Developer Portal on Kubernetes using Helm charts.

### Installation Options

There are two ways to install the portal on Kubernetes:

1. **As part of Tyk Self-Managed** - Enable `global.components.devPortal` during Tyk Self-Managed deployment using the [tyk-stack chart](/product-stack/tyk-charts/tyk-stack-chart)
2. **Standalone installation** - Use the `tyk-dev-portal` Helm chart (described below)

### Prerequisites

* [Kubernetes 1.19+](https://kubernetes.io/docs/setup/)
* [Helm 3+](https://helm.sh/docs/intro/install/)

### Installation Steps

1. **Create the `tyk-dev-portal-conf` secret**

    This secret is created automatically if Tyk Dashboard was bootstrapped with [tyk-bootstrap](https://artifacthub.io/packages/helm/tyk-helm/tyk-bootstrap) and `bootstrap.devPortal` was set to `true`.

    To create it manually:
    ```bash
    kubectl create secret generic tyk-dev-portal-conf -n ${NAMESPACE} \
    --from-literal=TYK_ORG=${TYK_ORG} \
    --from-literal=TYK_AUTH=${TYK_AUTH}
    ```

    The `TYK_ORG` and `TYK_AUTH` values are the Tyk Dashboard Organization ID and API Access Credentials, available in your Tyk Dashboard profile.

2. **Configure values.yaml**

    Set the following required values:

    | Field Name | Description |
    | ---------- | ----------- |
    | `global.adminUser.email` and `global.adminUser.password` | Portal admin credentials for bootstrapping |
    | `global.secrets.devPortal` | Secret name to enable portal bootstrapping |
    | `license` | Tyk license key |
    | `storage.type` | Portal storage type: `fs`, `s3`, or `db` |
    | `image.tag` | Portal version from [Docker Hub](https://hub.docker.com/r/tykio/portal/tags) |
    | `database.dialect` | Database type: `mysql` or `postgres` |
    | `database.connectionString` | Database connection string |

    Additional environment variables can be configured in the `extraEnvs` section. See the [configuration reference](/product-stack/tyk-enterprise-developer-portal/deploy/configuration).

3. **Install the portal**

    ```bash
    helm install tyk-dev-portal tyk-helm/tyk-dev-portal -f values.yaml -n tyk
    ```

For detailed configuration options, see the [Tyk Stack Helm chart documentation](/product-stack/tyk-charts/tyk-stack-chart).

<Note>
Helm chart supports Enterprise Portal v1.2.0 and later.
</Note>


## Linux (RPM)
<a id="linux-rpm"></a>

Install the portal on Red Hat Enterprise Linux, CentOS Stream, or compatible distributions using RPM packages.

<Warning>
**Operating System Requirements**

Portal v1.16.0 and later requires GLIBC 2.34+. This means:
- RHEL 9 or later is required
- CentOS Stream 9 or later is required
- RHEL 8 and CentOS 8 are NOT supported
</Warning>

**Prerequisites**
- RHEL 9, CentOS Stream 9, or compatible distribution
- Network connectivity to [packagecloud.io](https://packagecloud.io) (or download packages manually)
- PostgreSQL or MySQL database

### Installation Steps

1. **Download the portal package**

    Create an installation directory and download the package:
    ```console
    mkdir ~/portal-install
    cd ~/portal-install
    ```

    Download from [packagecloud.io](https://packagecloud.io/tyk/portal-unstable). Replace `<version>` with the desired version:
    ```console
    wget --content-disposition "https://packagecloud.io/tyk/portal-unstable/packages/el/9/portal-<version>.x86_64.rpm/download.rpm"
    ```

2. **Install the package**

    ```console
    sudo rpm -i portal-<version>.x86_64.rpm
    ```

3. **Configure the portal**

    Edit the configuration file at `/opt/portal/portal.conf`:
    ```json
    {
      "HostPort": 3001,
      "LicenseKey": "<your-license-here>",
      "Database": {
        "Dialect": "postgres",
        "ConnectionString": "host=localhost port=5432 dbname=portal user=admin password=secr3t sslmode=disable",
        "EnableLogs": false
      },
      "Storage": "fs",
      "TLSConfig": {
        "Enable": false
      }
    }
    ```

    See the [configuration reference](/product-stack/tyk-enterprise-developer-portal/deploy/configuration) for all options.

4. **Start the portal service**

    ```console
    sudo systemctl start portal.service
    ```

    Check the status:
    ```console
    systemctl status portal.service
    ```

5. **Bootstrap the portal**

    The portal is now running on port 3001. Complete the [bootstrapping process](#bootstrapping-enterprise-developer-portal) to create the super admin user.

## Bootstrapping Enterprise Developer Portal

When launching the portal for the first time, it starts in bootstrap mode to create the initial super admin user. You can bootstrap via the UI or the API.

### Bootstrapping via the UI

Navigate to the portal URL (e.g., `http://localhost:3001`) in your browser:

<img src="/img/dashboard/portal-management/enterprise-portal/portal-bootstrap-ui-empty.png" alt="Portal bootstrap UI" />

Enter the admin email, password, first name, and last name, then click **Register to Developer portal**.

<img src="/img/dashboard/portal-management/enterprise-portal/portal-bootstrap-successful.png" alt="Successful bootstrapping" />

Click **Login** to proceed to the login page.

### Bootstrapping via the API

To bootstrap programmatically:

```shell
curl --location 'http://<your-portal-host>:<your-portal-port>/portal-api/bootstrap' \
--header 'Content-Type: application/json' \
--data-raw '{
    "username":"super-admin@tyk.io",
    "password": "tyk123",
    "first_name":"John",
    "last_name":"Doe"
}'
```

Response:
```json
{
    "code": "OK",
    "message": "Bootstrapped user successfully",
    "data": {
        "api_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
}
```

<Note>
**Save the api_token**

You will need this token to call Portal APIs.
</Note>

### Logging in

After bootstrapping, use the super admin credentials to log in:

<img src="/img/dashboard/portal-management/enterprise-portal/navigate-to-the-login-page.png" alt="Open the login page" />

<img src="/img/dashboard/portal-management/enterprise-portal/login-page-after-bootstrapping.png" alt="Login page" />

<Note>
**Congratulations!**

Your portal is now ready. Continue configuring via the UI or the [Portal Admin API](/portal/overview/concepts).
</Note>
